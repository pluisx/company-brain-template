# 🧠 Memory

Chronological log of decisions, accumulated context, and business learnings.

## What This Is For

- Remember decisions made and why
- Accumulated context that AIs can read to understand the history
- Log of what worked and what didn't
- Ideas and observations you don't want to lose

## Entry Format

File name: `YYYY-MM-DD.md`

Suggested content:
```markdown
# [Date]

## Decisions Made
- [Decision] — Reason: [why]

## What Happened Today
- [Important event or action]

## Learnings
- [Something you learned or that worked/didn't work]

## Ideas to Explore
- [New idea that came up]

## Context for AIs
[Any information an AI should know about the state of your business on this date]
```

## How to Use with AIs

```
"Read all files in memory/ to understand the historical context of my business before helping me with [task]"
```

Or for recent context:
```
"Read memory/[recent date].md to understand what's happening right now"
```

## Tip: Make It a Habit

The memory folder is only useful if you update it. Try to write an entry:
- After important decisions
- When you close (or lose) a significant deal
- When you launch something new
- When you learn something that changes how you work
- At the end of each week as a mini-review

---

> 💡 AIs don't have memory between sessions. These files are your continuity layer.
